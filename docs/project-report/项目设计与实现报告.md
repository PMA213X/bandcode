# 熏掌门 AI 智能客服与业务助理系统 — 设计与实现报告

## 一、项目概述

### 1.1 项目背景

熏掌门是武夷山非遗食品品牌，主要销售非遗熏鹅产品。目前主要通过微信、抖音、小红书等渠道进行销售。在用户咨询过程中，大量问题高度重复（口味选择、储存方式、保质期、加热方式、物流时效等），人工客服重复回答效率低下，且无法保证回答的一致性。

同时，团队在日常运营中也面临信息分散、知识难以沉淀的问题——产品文档、客服话术、运营策略散落在不同平台，缺乏统一的知识管理和智能检索手段。

基于以上痛点，项目决定构建一套 AI 智能客服与业务助理系统，通过 RAG（检索增强生成）技术结合多 Agent 协作架构，实现自动问答、知识库管理、任务编排等功能。

### 1.2 项目目标

1. **智能问答**：基于大模型与 RAG 技术，实现 7x24 小时自动回答用户常见问题
2. **知识库管理**：构建结构化知识库，支持文档切分、向量化索引和语义检索
3. **多 Agent 协作**：通过自研 Pipeline 工作流编排多个专业 Agent，实现需求分析、代码生成、测试验证、约束审查的自动化流程
4. **工具调用**：Agent 可调用文件读写、目录浏览、知识库搜索、任务管理等内置工具
5. **分层记忆**：实现六层 Memory 系统（global / project / task / session / checkpoint / notes），支持上下文持久化和跨会话知识积累
6. **实时交互**：通过 SSE（Server-Sent Events）实现流式响应，前端实时展示 Agent 执行状态

### 5.3 关键实现

#### 5.3.1 文档切分器（DocumentChunker）

文档切分器负责将原始文档拆分为适合向量化的小段落。支持两种切分策略：

- **按段落切分**：以 `\n\n` 为分隔符，适用于结构化 Markdown 文档
- **按句子切分**：以中英文句号、感叹号、问号为分隔符，适用于连续文本

切分参数：
- `max_chunk_size`：单个 chunk 最大长度，默认 1000 字符
- `overlap`：相邻 chunk 的重叠字符数，默认 100 字符

```python
# backend/rag/chunker.py
@dataclass
class TextChunk:
    """文本块数据结构"""
    content: str       # 文本内容
    index: int         # 块序号
    metadata: dict = None  # 元数据（段落号、子索引等）

class DocumentChunker:
    def __init__(self, max_chunk_size: int = 1000, overlap: int = 100):
        self.max_chunk_size = max_chunk_size
        self.overlap = overlap

    def chunk_by_paragraph(self, text: str) -> list[TextChunk]:
        """按段落切分文档"""
        paragraphs = [p.strip() for p in text.split("\n\n") if p.strip()]
        chunks = []
        for i, para in enumerate(paragraphs):
            if len(para) > self.max_chunk_size:
                sub_chunks = self._split_long_text(para)
                for j, sub in enumerate(sub_chunks):
                    chunks.append(TextChunk(
                        content=sub,
                        index=len(chunks),
                        metadata={"paragraph": i, "sub_index": j}
                    ))
            else:
                chunks.append(TextChunk(
                    content=para, index=i, metadata={"paragraph": i}
                ))
        return chunks
```

对于超长段落（超过 `max_chunk_size`），切分器会尝试在句号处断开，并保留 overlap 以维持上下文连贯性。

#### 5.3.2 向量化客户端（EmbeddingClient）

基于 SentenceTransformers 封装，支持单条和批量向量化：

```python
# backend/models/embedding.py
class EmbeddingClient:
    """sentence-transformers 的 Embedding 封装"""

    MODEL_DIMENSIONS = {
        "all-MiniLM-L6-v2": 384,
        "all-mpnet-base-v2": 768,
        "paraphrase-multilingual-MiniLM-L12-v2": 384,
    }

    def __init__(self, model_name: str = "all-MiniLM-L6-v2"):
        self.model_name = model_name
        self.model = SentenceTransformer(model_name)
        self.dimension = self.MODEL_DIMENSIONS.get(model_name, 384)

    def embed(self, text: str) -> list[float]:
        """单条文本向量化"""
        embedding = self.model.encode(text, convert_to_numpy=True)
        return embedding.tolist()

    def embed_batch(self, texts: list[str]) -> list[list[float]]:
        """批量文本向量化"""
        embeddings = self.model.encode(texts, convert_to_numpy=True)
        return embeddings.tolist()
```

默认使用 `all-MiniLM-L6-v2` 模型，生成 384 维向量。该模型支持中英文混合文本，推理速度快，适合实时检索场景。

#### 5.3.3 RAG 索引器（RAGIndexer）

负责文档的切分、向量化和 ChromaDB 入库：

```python
# backend/rag/indexer.py
class RAGIndexer:
    def __init__(self, chroma_path: str = "./chroma",
                 collection_name: str = "knowledge",
                 embedding_client: Optional[EmbeddingClient] = None,
                 chunker: Optional[DocumentChunker] = None):
        self.embedding_client = embedding_client or EmbeddingClient()
        self.chunker = chunker or DocumentChunker()
        # 初始化 ChromaDB 持久化客户端
        self.client = chromadb.PersistentClient(path=chroma_path)
        self.collection = self.client.get_or_create_collection(
            name=collection_name,
            metadata={"hnsw:space": "cosine"}  # 使用余弦距离
        )

    def index_documents(self, documents: list[str],
                        metadatas: Optional[list[dict]] = None) -> int:
        """索引多条文档，返回索引的 chunk 数量"""
        all_chunks = []
        for i, doc in enumerate(documents):
            meta = metadatas[i] if metadatas else {"source": f"doc_{i}"}
            chunks = self.chunker.chunk_by_paragraph(doc)
            for chunk in chunks:
                chunk.metadata.update(meta)
                all_chunks.append(chunk)
        # 批量向量化并存入 ChromaDB
        texts = [c.content for c in all_chunks]
        embeddings = self.embedding_client.embed_batch(texts)
        ids = [f"chunk_{self.collection.count() + i}" for i in range(len(all_chunks))]
        self.collection.add(ids=ids, documents=texts,
                            embeddings=embeddings, metadatas=all_metadatas)
        return len(all_chunks)
```

支持三种索引模式：单条文档、文件、目录（按 glob 模式匹配）。

#### 5.3.4 RAG 检索器（RAGRetriever）

基于 ChromaDB 的相似度检索，返回带有相关性评分的结果：

```python
# backend/rag/retriever.py
class RAGRetriever:
    def search(self, query: str, top_k: int = 5) -> list[SearchResult]:
        """相似度检索，返回最相关的 top_k 个结果"""
        query_embedding = self.embedding_client.embed(query)
        results = self.collection.query(
            query_embeddings=[query_embedding],
            n_results=min(top_k, self.collection.count())
        )
        search_results = []
        if results and results["documents"]:
            for i, doc in enumerate(results["documents"][0]):
                score = 1 - results["distances"][0][i]  # 余弦距离转相似度
                metadata = results["metadatas"][0][i] if results["metadatas"] else {}
                search_results.append(SearchResult(
                    content=doc, score=score, metadata=metadata
                ))
        return search_results
```

检索器还支持带过滤条件的检索（`search_with_filter`），可通过 ChromaDB 的 `where` 参数按元数据过滤结果。

## 六、Agent 与工具调用设计

### 6.1 Agent 架构

系统采用基于 `BaseAgent` 抽象基类的 Agent 架构，由 `AgentManager` 统一管理所有 Agent 的注册、发现和执行。当前共包含 6 个专业 Agent：

| Agent | 名称 | 职责 | 模型 | 权限 |
|-------|------|------|------|------|
| PlannerAgent | planner | 需求分析、任务拆解、Agent 调度 | mimo-v2.5-pro | read/edit/bash |
| SimpleCoderAgent | simple-coder | 简单编码（UI 修改、配置调整、小型 Bug） | mimo-v2.5 | read/edit/bash |
| ComplexCoderAgent | complex-coder | 复杂编码（核心逻辑、API 开发、架构调整） | mimo-v2.5-pro | read/edit/bash |
| TesterAgent | tester | 编译检查、单元测试、静态分析 | mimo-v2.5 | read（无 edit 权限） |
| ConstraintAgent | constraint | 从 Memory 中智能检索相关约束 | mimo-v2.5 | read（无 edit/bash 权限） |
| ReviewAgent | review | 检查输出是否违反项目约束 | mimo-v2.5 | read（无 edit/bash 权限） |

#### 6.1.1 BaseAgent 基类

所有 Agent 继承 `BaseAgent`，提供统一的 LLM 调用、Tool 调用、状态上报接口：

```python
# backend/agents/base.py
@dataclass
class PipelineState:
    """贯穿整个工作流的状态数据结构"""
    user_input: str = ""          # 用户输入
    session_id: str = ""          # 会话 ID
    project: str = ""             # 项目名称
    constraints: list[str] = field(default_factory=list)  # 约束列表
    constraint_summary: str = ""  # 约束摘要
    rag_context: str = ""         # RAG 检索上下文
    memory_context: dict = field(default_factory=dict)    # Memory 上下文
    plan: dict = field(default_factory=dict)              # 规划结果
    code: str = ""                # 生成的代码
    files_changed: list[dict] = field(default_factory=list)
    test_results: dict = field(default_factory=dict)
    review_result: dict = field(default_factory=dict)
    agent_history: list[dict] = field(default_factory=list)
    error: Optional[str] = None
    current_agent: str = ""
    step_count: int = 0
    max_steps: int = 50

class BaseAgent(ABC):
    """Agent 基类，所有 Agent 必须实现 run() 方法"""
    name: str = "base"
    description: str = "基础 Agent"
    model: str = "mimo-v2.5-pro"
    temperature: float = 0.1
    permissions: dict = field(default_factory=lambda: {
        "read": "allow", "edit": "allow", "bash": "allow"
    })

    def __init__(self, llm_client: LLMClient):
        self.llm = llm_client
        self._status_callback: Optional[Callable] = None
        self._tool_registry = None

    @abstractmethod
    async def run(self, state: PipelineState) -> PipelineState:
        """执行 Agent 逻辑（子类必须实现）"""
        raise NotImplementedError

    async def call_tool(self, tool_name: str, args: dict) -> Any:
        """通过 ToolRegistry 调用工具"""
        result = await self._tool_registry.call(tool_name, args, self.permissions)
        return result

    def report_status(self, status: str, data: dict = None):
        """上报状态（用于 SSE 推送）"""
        if self._status_callback:
            self._status_callback({"agent": self.name, "status": status, "data": data or {}})
```

`PipelineState` 是贯穿整个工作流的状态数据结构，每个 Agent 接收 state 并返回修改后的 state，通过状态传递实现节点间通信。

#### 6.1.2 AgentManager 管理器

`AgentManager` 负责 Agent 的自动发现、注册和执行：

```python
# backend/agents/manager.py
class AgentManager:
    def __init__(self, llm_client: LLMClient):
        self.llm_client = llm_client
        self.agents: dict[str, BaseAgent] = {}

    def auto_discover(self, agents_dir: str = None):
        """自动扫描 agents/ 目录，注册所有 Agent"""
        for file_path in Path(agents_dir).glob("*.py"):
            if file_path.name.startswith("_") or file_path.name in ["base.py", "manager.py"]:
                continue
            module = importlib.import_module(f"agents.{file_path.stem}")
            for attr_name in dir(module):
                attr = getattr(module, attr_name)
                if (isinstance(attr, type) and issubclass(attr, BaseAgent)
                        and attr is not BaseAgent):
                    agent = attr(self.llm_client)
                    self.register(agent)

    async def run(self, name: str, state: PipelineState) -> PipelineState:
        """执行指定 Agent"""
        agent = self.get(name)
        if agent is None:
            raise ValueError(f"Agent not found: {name}")
        if not self._check_permissions(agent):
            raise PermissionError(f"Agent {name} lacks required permissions")
        state.current_agent = name
        return await agent.run(state)
```

`auto_discover` 方法通过动态导入和反射，自动扫描 `agents/` 目录下所有继承 `BaseAgent` 的子类并注册，实现 Agent 的即插即用。

#### 6.1.3 PlannerAgent 规划调度

PlannerAgent 是整个工作流的调度中心，负责需求分析、任务拆解和子 Agent 选择：

```python
# backend/agents/planner.py
class PlannerAgent(BaseAgent):
    name = "planner"
    description = "规划调度智能体"
    model = "mimo-v2.5-pro"
    temperature = 0.3

    async def run(self, state: PipelineState) -> PipelineState:
        # 1. 需求分析：LLM 解析用户意图、复杂度、影响范围
        analysis = await self._analyze_requirement(state)
        # 2. 任务拆解：将需求分解为可执行的子任务列表
        tasks = await self._decompose_tasks(analysis, state)
        # 3. 选择子 Agent：根据任务复杂度分配 simple-coder / complex-coder / tester
        delegated_agent, reason = self._select_agent(tasks, analysis)
        # 4. 输出计划
        state.plan = {
            "tasks": tasks,
            "delegated_agent": delegated_agent,
            "reason": reason,
            "analysis": analysis
        }
        return state
```

Agent 选择规则：
- **simple-coder**：UI 修改、配置调整、单文件修改、小型 Bug 修复（低复杂度）
- **complex-coder**：核心业务逻辑、API 开发、架构调整、跨模块重构（高复杂度）
- **tester**：编译检查、单元测试、静态分析

#### 6.1.4 ConstraintAgent 约束检索

ConstraintAgent 从六层 Memory 中智能检索与当前问题相关的约束：

```python
# backend/agents/constraint.py
class ConstraintAgent(BaseAgent):
    name = "constraint"
    description = "约束检索智能体"
    model = "mimo-v2.5"
    temperature = 0.1
    permissions = {"read": "allow", "edit": "deny", "bash": "deny"}

    async def run(self, state: PipelineState) -> PipelineState:
        # 1. 使用 LLM 解析用户意图关键词
        intent = await self._parse_intent(state.user_input)
        # 2. 从 Memory 各层检索相关约束
        all_constraints = await self._search_constraints(intent, state)
        # 3. 去重 + 按相关性排序
        unique_constraints = self._deduplicate(all_constraints)
        ranked_constraints = self._rank_by_relevance(unique_constraints, intent)
        # 4. 取 top_k 并生成摘要
        top_constraints = ranked_constraints[:self.max_constraints]
        summary = await self._generate_summary(top_constraints)
        state.constraints = top_constraints
        state.constraint_summary = summary
        return state
```

约束检索基于分词交集判断相关性：将用户意图和约束内容分别分词（支持中英文混合），计算交集大小作为相关性得分。

#### 6.1.5 ReviewAgent 约束审查

ReviewAgent 在代码生成后检查输出是否违反项目约束：

```python
# backend/agents/review.py

    model = "mimo-v2.5"
    temperature = 0
```

- **文件合规**：检查是否修改了禁止修改的敏感文件（settings.json、.env、*.key 等）

#### 6.2.1 Tool 基类

```python
# backend/tools/base.py
@dataclass
class ToolResult:
    """工具执行结果"""
    success: bool
    data: Any = None
    error: Optional[str] = None
    execution_time: Optional[float] = None

class Tool(ABC):
    """工具基类，所有工具必须实现 execute() 方法"""
    name: str = "base"
    description: str = "基础工具"
    permission: str = "read"  # read / write / bash / admin
    parameters: dict = field(default_factory=dict)

    @abstractmethod
    async def execute(self, **kwargs) -> ToolResult:
        raise NotImplementedError

    def get_schema(self) -> dict:
        return {"name": self.name, "description": self.description,
                "parameters": self.parameters}
```

#### 6.2.2 ToolRegistry 注册中心

```python
# backend/tools/registry.py
class ToolRegistry:
    def __init__(self):
        self.tools: dict[str, Tool] = {}

    def auto_discover(self, tools_dir: str = None):
        """自动扫描 tools/ 目录和 builtins/ 子目录，注册所有 Tool"""
        # 扫描 tools/*.py
        for file_path in Path(tools_dir).glob("*.py"):
            # ... 动态导入并注册
            pass
        # 扫描 tools/builtins/*.py
        builtins_dir = Path(tools_dir) / "builtins"
        if builtins_dir.exists():
            self._discover_builtins(builtins_dir)

    async def call(self, name: str, args: dict,
                   agent_permissions: dict) -> ToolResult:
        """调用工具：获取工具 → 权限校验 → 参数验证 → 执行"""
        tool = self.get(name)
        if tool is None:
            return ToolResult(success=False, error=f"Tool not found: {name}")
        if not self._check_permission(tool, agent_permissions):
            return ToolResult(success=False, error="Permission denied")
        if not tool.validate_params(**args):
            return ToolResult(success=False, error="Invalid parameters")
        result = await tool.execute(**args)
        return result
```

工具调用流程：`获取工具实例 → 权限校验 → 参数验证 → 执行 → 返回 ToolResult`。权限校验基于 Agent 的 `permissions` 字典与工具所需 `permission` 的匹配。

#### 6.3.2 Pipeline 8 节点管线

```python
# backend/workflow/pipeline.py
class Pipeline:
    """主工作流管线 — 8 个节点顺序执行"""

    def __init__(self, config: dict = None):
        self.config = config or DEFAULT_CONFIG.copy()
        self.nodes: list[Callable] = [
            self.node_constraint,      # 节点 1: 约束检索
            self.node_rag,             # 节点 2: RAG 检索
            self.node_prompt_build,    # 节点 3: Prompt 构建
            self.node_planner,         # 节点 4: Planner 调度
            self.node_approval,        # 节点 5: 审批检查
            self.node_subagent,        # 节点 6: 子 Agent 执行
            self.node_tester,          # 节点 7: Tester 验证
            self.node_review,          # 节点 8: Review 审查
        ]

    async def run(self, state: PipelineState) -> PipelineState:
        """执行主管线"""
        for node in self.nodes:
            if state.done or state.error:
                break
            try:
                state = await node(state)
            except Exception as e:
                state.error = f"节点 {node.__name__} 执行失败: {str(e)}"
                break
        return state

    async def run_with_review_loop(self, state: PipelineState) -> PipelineState:
        """执行主管线（带 Review 修正循环）"""
        for attempt in range(state.max_retries):
            state = await self.run()
            if (state.review_result
                    and not state.review_result.get("passed", True)
                    and self.config.get("自动修正", True)):
                state.retry_count += 1
                # 将违规信息反馈给 Planner 重新生成
                violations = state.review_result.get("violations", [])
                state.user_input = "请修正以下违规项：\n" + "\n".join(
                    f"- {v.get('detail', '')}" for v in violations
                )
                continue
            break
        return state
```

```python
class LLMClient:
    """OpenAI 兼容的 LLM 客户端封装"""

    def __init__(self, base_url: str, api_key: str, model: str,
                 timeout: int = 60, max_retries: int = 3):
        self.client = AsyncOpenAI(
            base_url=base_url, api_key=api_key,
            timeout=timeout, max_retries=max_retries
        )

    async def chat_stream(self, messages: list[dict], temperature: float = 0.1,
                          max_tokens: Optional[int] = None) -> AsyncGenerator[str, None]:
        """流式对话，逐块返回响应"""
        stream = await self.client.chat.completions.create(
            model=self.model, messages=messages,
            temperature=temperature, stream=True
        )
        async for chunk in stream:
            if chunk.choices and chunk.choices[0].delta.content:
                yield chunk.choices[0].delta.content
```

错误处理涵盖认证失败（401）、端点不存在（404）、频率超限（429）、服务器错误（500）等场景，将 API 错误转换为用户友好的中文提示。

```python
class MemoryStore:
    """六层分层 Memory 存储管理器

    层级:
    - global:    全局偏好、编码规范、技术栈偏好（更新频率低）
    - project:   架构决策、模块说明、项目约定（更新频率中）

    - session:   对话历史摘要（每会话更新）
    - checkpoint: 文件变更列表、快照摘要（按需更新）
    - notes:     TODO、临时记录、灵感（任意更新）
    """

    def __init__(self, project_path: str):
        self.base_path = Path(project_path) / ".mimo"

    def get_memory(self, layer: str, session_id: str = None) -> str:
        """读取指定层级的 Memory"""
        if layer == "global":
            return self._read_file(self.base_path / "global" / "MEMORY.md")
        elif layer == "project":
            return self._read_file(self.base_path / "project" / "MEMORY.md")
        # ... 其他层级

    def search_all_layers(self, query: str) -> dict[str, list[str]]:
        """在所有层级中搜索"""
        results = {}
        for layer in ["global", "project", "task", "notes"]:
            matches = self.search_memory(layer, query)
            if matches:
                results[layer] = matches
        return results
```

## 九、总结与展望

### 9.1 项目总结

本项目成功实现了基于自研 Pipeline 工作流与多 Agent 协作架构的 AI 智能客服与业务助理系统，主要成果包括：

1. **自研 Pipeline 工作流引擎**：8 节点顺序管线 + Review 修正循环，通过 `PipelineState` 数据类实现节点间状态传递，替代了 LangGraph 等重型框架，保持了架构的简洁性和可控性。

2. **六 Agent 协作架构**：基于 `BaseAgent` 抽象基类和 `AgentManager` 自动发现机制，实现了 Planner、SimpleCoder、ComplexCoder、Tester、Constraint、Review 六个专业 Agent 的即插即用和权限隔离。

3. **RAG 知识库**：采用 `DocumentChunker` + `EmbeddingClient`（SentenceTransformers all-MiniLM-L6-v2）+ `ChromaDB` 的技术栈，实现了文档切分、向量化索引和语义检索的完整流程。

4. **分层 Memory 系统**：六层分层存储（global / project / task / session / checkpoint / notes），以 Markdown 文件形式持久化，支持跨会话知识积累和上下文管理。

5. **实时流式交互**：通过 SSE（Server-Sent Events）实现 12 种事件类型的实时推送，前端使用 React `EventSource` API 逐字显示 AI 回复。

6. **完善的工具系统**：8 个内置工具，支持自动发现、权限校验和参数验证，Agent 可通过统一接口调用文件操作、搜索、任务管理等能力。

7. **全面的自动化测试**：覆盖 RAG、Agent、Tool、Workflow、API 五个模块，使用 pytest + pytest-asyncio 进行异步测试。
