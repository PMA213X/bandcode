# 第六章 小组团队分工

## 6.1 团队成员与角色

| 成员 | 角色 | 主要职责 |
|------|------|----------|
| 成员A | 组长 / 项目经理 | 项目架构设计、工作流引擎、配置管理、错误处理、中间件 |
| 成员B | AI 开发工程师 A（RAG 方向） | RAG 知识库设计与实现、文本切分、向量化、语义检索 |
| 成员C | AI 开发工程师 B（Agent 方向） | Agent 系统设计与实现、工具系统、提示词管理 |
| 成员D | 后端开发工程师 A | API 设计与实现、SSE 事件系统、LLM 客户端、数据模型 |
| 成员E | 后端开发工程师 B | 数据库设计、记忆系统、项目管理 API、工具 API |
| 成员F | 前端开发工程师 A | 聊天界面、侧边栏、命令面板、设置界面 |
| 成员G | 前端开发工程师 B | 历史记录、记忆视图、模型测试、文件浏览器、工作区信息 |

## 6.2 成员A — 组长 / 项目经理

### 角色定位

负责项目整体架构设计，主导工作流引擎和配置系统的实现，确保各模块之间的协调与集成。

### 负责的源代码文件

| 文件 | 功能说明 |
|------|----------|
| `backend/main.py` | FastAPI 应用入口，注册 8 组路由、CORS 配置、中间件挂载、全局异常处理器 |
| `backend/config/loader.py` | ConfigLoader 单例，JSON 配置文件读取，6 个配置分区（模型/Agent/Memory/Workflow/RAG/Tool） |
| `backend/config/logging.py` | ChineseFormatter 中文日志格式化、setup_logging 日志初始化、get_logger 获取日志器 |
| `backend/workflow/state.py` | PipelineState 数据结构，17 个字段贯穿整个 Pipeline 执行过程 |
| `backend/workflow/pipeline.py` | Pipeline 工作流引擎，8 节点顺序执行（约束检索→RAG→Prompt→Planner→审批→子Agent→Tester→Review） |
| `backend/workflow/review_loop.py` | ReviewLoop 审查循环，自动修复重试机制，最大重试次数控制 |
| `backend/workflow/checkpoint.py` | CheckpointManager 快照管理器，文件快照创建、恢复、列表、删除 |
| `backend/api/errors.py` | ErrorCode 错误码枚举、AppException 自定义异常、register_error_handlers 全局异常处理器 |
| `backend/api/middleware.py` | RequestLoggingMiddleware 请求日志中间件、SecurityHeadersMiddleware 安全头中间件 |

### 所需知识点

- FastAPI 应用生命周期管理（startup/shutdown 事件）
- ASGI 中间件编写（BaseHTTPMiddleware）
- Python 设计模式（单例模式、模板方法模式、策略模式）
- 异步编程（async/await、asyncio 任务管理）
- 项目架构设计（分层架构、模块解耦）
- JSON 配置文件管理与热重载

## 6.3 成员B — AI 开发工程师 A（RAG 方向）

### 角色定位

负责 RAG（Retrieval-Augmented Generation）知识库的设计与实现，包括文档切分、向量化、索引和检索。

### 负责的源代码文件

| 文件 | 功能说明 |
|------|----------|
| `backend/rag/chunker.py` | DocumentChunker 文本切分器，TextChunk 数据结构，chunk_by_paragraph 按段落切分，chunk_by_sentence 按句子切分，_split_long_text 超长文本切分 |
| `backend/rag/indexer.py` | RAGIndexer 索引器，ChromaDB PersistentClient 初始化，index_documents/index_file/index_directory 三级索引接口 |
| `backend/rag/retriever.py` | RAGRetriever 检索器，search 相似度检索，search_with_filter 带过滤条件检索，SearchResult 数据结构 |
| `backend/models/embedding.py` | EmbeddingClient 向量化封装，SentenceTransformer 模型加载，embed/embed_batch 单条和批量向量化，MODEL_DIMENSIONS 多模型维度映射 |

### 所需知识点

- ChromaDB 向量数据库（PersistentClient、Collection、cosine 距离、HNSW 索引）
- SentenceTransformers 文本向量化（all-MiniLM-L6-v2 模型，384 维向量）
- 文本切分策略（按段落切分、按句子切分、overlap 重叠机制）
- 余弦相似度检索与 Top-K 策略
- Python dataclass 与类型注解
- pathlib 文件路径处理

## 6.5 成员D — 后端开发工程师 A

### 角色定位

| `backend/api/sse.py` | SSE 事件系统，SSEEventType 12 种事件类型枚举、SSEEvent 事件模型、ConnectionManager 连接管理器、sse_generator 事件生成器、11 个 push_* 推送函数 |

### 所需知识点
